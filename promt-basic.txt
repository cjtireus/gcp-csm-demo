Act as a Google Cloud Solutions Architect. Your goal is to generate a complete bash script named csm-demo-script-basic.sh and the necessary Kubernetes manifests to deploy a proxy-less gRPC service mesh on GKE Autopilot.

CRITICAL CONSTRAINTS:
1. USE ONLY KUBERNETES GATEWAY API (GAMMA): Do not use Istio CRDs (VirtualService, etc.) or legacy Traffic Director APIs. Never try to use TDMesh. 2. PROXY-LESS ONLY: Use gRPC native xDS. No Envoy sidecars.
3. IDENTITY: Use native GKE Workload Identity (Project-level principal), not custom GSAs.
4. Use Namespace name 'csm-demo-basic'

1. Enable APIs: cloudresourcemanager, compute, container, networkservices, networksecurity, certificatemanager, privateca, meshconfig, trafficdirector, gkehub.

    - Grant `roles/anthosservicemesh.serviceAgent`, `roles/meshcontrolplane.serviceAgent`, and `roles/compute.securityAdmin` to the Cloud Service Mesh Service Agent (`service-${PROJECT_NUMBER}@gcp-sa-servicemesh.iam.gserviceaccount.com`).
3. Create VPC and Subnet in europe-north2.
4. Provision GKE Autopilot cluster 'csm-autopilot-cluster' with --enable-mesh-certificates and --gateway-api=standard. Follow 'Prepare to setup the Gateway API for Cloud Service Mesh', https://docs.cloud.google.com/service-mesh/docs/gateway/prepare-gateway
5. Mesh Config: Register the cluster to the Fleet and enable 'servicemesh' with '--config-api gateway'.
6. IAM: Bind 'privateca.certificateRequester' to the GKE service agents and grant 'roles/trafficdirector.client' to the member 'group:${PROJECT_ID}.svc.id.goog:/allAuthenticatedUsers/'.

    - Create a Namespace with 'mesh.cloud.google.com/csm-injection=proxyless' and `mesh.cloud.google.com/managed-csm="true"`.
    - Create an HTTPRoute (GAMMA) for a 'greeter' service. Ensure the `parentRefs` refers to the `Service` and includes the correct port.
    - Create ServerTlsPolicy (allowOpen: false) and ClientTlsPolicy.
    - Create a Global Endpoint Policy matching the label `app: greeter` and attach the Server TLS Policy to it.
    - Create a server that allows me to test the mesh with grpcurl.
    - Create a tester pod that allows me to test the mesh with grpcurl. Use annotation `security.cloud.google.com/use-workload-certificates: ""`.

10. Exec into the server pod and print the `GRPC_XDS_BOOTSTRAP` environment variable to verify proxyless injection. Also print the content of the `td-grpc-bootstrap.json` file.     
11. Validation: Run a 'grpcurl' command that uses the `xds:///` with plaintext to test the connection to the server pod. Validate the response for success and print the outcome.
12. Validation: Run a 'grpcurl' command that uses the `xds:///` with mtls to test the connection to the server pod. Validate the response for success and print the outcome.
